یک عدد صحیح (بین ۱ تا ۳۹۹۹) را به رشته‌ی عدد رومی آن تبدیل کن. اعداد رومی از هفت نماد استفاده
می‌کنند و یک قانون تفریق دارند: وقتی یک نماد کوچک‌تر باید چهار بار تکرار شود، به‌جای آن یک جفت
تفریقی استفاده می‌شود.

### نمادهای اعداد رومی

| نماد | مقدار |
| --- | --- |
| I | ۱ |
| V | ۵ |
| X | ۱۰ |
| L | ۵۰ |
| C | ۱۰۰ |
| D | ۵۰۰ |
| M | ۱۰۰۰ |

شش حالت تفریقی عبارتند از: **IV** = ۴، **IX** = ۹، **XL** = ۴۰، **XC** = ۹۰، **CD** = ۴۰۰، **CM** = ۹۰۰.

### جدول‌های جستجوی رقمی

چون یک عدد رومی هیچ‌گاه نمادهای دو «دهه» را با هم قاطی نمی‌کند، هر رقم از `num` (هزارگان، صدگان،
دهگان، یکان) را می‌توان مستقل با یک جدول جستجوی ثابت با ۱۰ ورودی — یکی برای هر مقدار رقم ۰ تا ۹ —
تبدیل کرد. به‌هم‌پیوستن چهار جستجو به‌ترتیب، رشته‌ی نهایی را می‌سازد.

```
1994
 رقم هزارگان 1 → "M"
 رقم صدگان   9 → "CM"
 رقم دهگان   9 → "XC"
 رقم یکان    4 → "IV"
 "M" + "CM" + "XC" + "IV" = "MCMXCIV"
```

### رویکرد جایگزین: حریصانه

فهرستی از مقادیر را از بزرگ به کوچک پیمایش کن، شامل شش جفت تفریقی (۱۰۰۰، ۹۰۰، ۵۰۰، ۴۰۰، ۱۰۰، ۹۰،
۵۰، ۴۰، ۱۰، ۹، ۵، ۴، ۱). برای هر مقدار، تا جایی که هنوز در `num` جا می‌شود نمادش را اضافه کن و آن
را کم کن، سپس به مقدار کوچک‌تر بعدی برو.

```
1994
 1000 یک‌بار جا می‌شود → "M",  num = 994
  900 یک‌بار جا می‌شود → "CM", num = 94
   90 یک‌بار جا می‌شود → "XC", num = 4
    4 یک‌بار جا می‌شود → "IV", num = 0
 "MCMXCIV"
```

### پیچیدگی

| روش | زمان | حافظه |
| --- | --- | --- |
| جدول‌های جستجوی رقمی | O(1) | O(1) |
| حریصانه | O(1) | O(1) |

هر دو روش زمان/حافظه ثابتی دارند چون `num` به ۳۹۹۹ محدود است — جدول‌های جستجو اندازه‌ی ثابتی دارند
و حلقه‌ی حریصانه حداکثر ۱۳ مقدار را بررسی می‌کند.

### توضیح برای یک بچه 🧒

فرض کن یک جعبه استیکر داری با برچسب‌های M، D، C، L، X، V، I — هرکدام ارزش امتیاز متفاوتی دارند.
می‌خواهی یک عدد را با کمترین تعداد استیکر بسازی.

یک راه: **بزرگ‌ترین** استیکری که هنوز زیر عددت جا می‌شود را بردار، بچسبان و امتیازش را کم کن. تا
وقتی به صفر برسی همین کار را با بزرگ‌ترین استیکر مناسب تکرار کن — این راه حریصانه است.

راه دیگر: عددت را به «چند هزارگان، چند صدگان، چند دهگان، چند یکان» تقسیم کن و برای هر بخش، ترکیب
آماده‌ی استیکرها را از یک برگه‌ی تقلب پیدا کن — اصلاً نیازی به کم کردن نیست!
