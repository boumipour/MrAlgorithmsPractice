عنصری را برگردانید که **بیش از ⌊n / 2⌋ بار** ظاهر می‌شود. مسئله تضمین می‌کند که چنین عنصری وجود دارد.

### رأی‌گیری بویر–مور (Boyer–Moore)

یک `candidate` و یک `count` نگه می‌داریم:

- وقتی `count == 0` شد، مقدار فعلی را به‌عنوان `candidate` جدید می‌پذیریم.
- اگر مقدار فعلی برابر کاندیدا باشد `count++`، در غیر این‌صورت `count--`.

هر مقدار غیرکاندیدا یک رأی کاندیدا را خنثی می‌کند. چون اکثریت واقعی بیش از نیمی از مواقع رخ می‌دهد،
هرگز به‌طور کامل خنثی نمی‌شود — پس به‌عنوان `candidate` نهایی باقی می‌ماند.

این روش از نظر حافظه از شمارش با جدول درهم‌سازی بهتر است: **O(1)** به‌جای O(n).

### پیچیدگی

- **زمان:** O(n) — یک پیمایش.
- **حافظه:** O(1).

### توضیح برای یک بچه 🧒

بیا برنده‌ی `[2, 2, 1, 1, 1, 2, 2]` را بشماریم. یک **رهبر** و یک **امتیاز** که از ۰ شروع می‌شود نگه دار.

۱. `2` را ببین، امتیاز ۰ است → `2` را رهبر کن، امتیاز = ۱.
۲. `2` را ببین، با رهبر یکی است → امتیاز = ۲.
۳. `1` را ببین، متفاوت → امتیاز = ۱.
۴. `1` را ببین، متفاوت → امتیاز = ۰.
۵. `1` را ببین، امتیاز ۰ است → `1` را رهبر کن، امتیاز = ۱.
۶. `2` را ببین، متفاوت → امتیاز = ۰.
۷. `2` را ببین، امتیاز ۰ است → `2` را رهبر کن، امتیاز = ۱.

رهبری که در پایان سرِپا مانده `2` است — و واقعاً هم `2` بیش از نیمی از مواقع ظاهر می‌شود. 🏆
